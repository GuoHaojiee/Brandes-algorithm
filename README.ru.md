# Центральность по посредничеству Брандеса · Параллельная реализация MPI + CUDA

> [阅读中文版](README.md)

Распределённая GPU-реализация **алгоритма Брандеса** для вычисления центральности по посредничеству,  
предназначенная для суперкомпьютерного кластера IBM Polus (архитектура ppc64le).  
**MPI** распределяет задачи по узлам, **CUDA** выполняет BFS и обратное распространение на GPU.

---

## Среда выполнения

| Параметр | Значение |
|----------|----------|
| Кластер | IBM Polus (ppc64le) |
| GPU | NVIDIA Tesla P100 16 ГБ × 2 (соединение NVLink) |
| CUDA | 10.2, архитектура sm\_60 |
| MPI | `mpi/openmpi3-ppc64le` |
| Компиляторы | `nvcc` + `mpicxx` |

## Быстрый старт

```bash
# 1. Загрузить модуль MPI
module load mpi/openmpi3-ppc64le

# 2. Собрать проект
make

# 3. Запустить (2 процесса MPI, каждый привязан к одному P100)
mpirun -np 2 ./brandes test.txt > out.txt

# 4. Проверить результат (один из вариантов)
python3 verify.py test.txt out.txt          # требует networkx
python3 verify_stdlib.py test.txt out.txt   # без сторонних зависимостей

# Или через конвейер (pipeline)
mpirun -np 2 ./brandes test.txt | python3 verify_stdlib.py test.txt
```

## Формат входных данных

Файл графа — список рёбер, по одному ребру на строку. Строки с `#` считаются комментариями:

```
# Комментарий
0 1
1 2
2 3
```

Нумерация вершин начинается с `0`; общее число вершин определяется автоматически.

---

## Формат вывода

Стандартный вывод (stdout) — центральность каждой вершины:

```
0 0.000000
1 3.500000
2 5.000000
...
```

Хронометраж выводится в стандартный поток ошибок (stderr) и не мешает передаче данных через конвейер:

```
计算时间（GPU BFS+反向传播）: 0.0591 秒
通信时间（MPI_Reduce）:        0.0002 秒
```

## Описание алгоритма

### Общая схема

```
rank 0 читает граф → MPI_Bcast рассылает данные → каждый процесс получает свой набор истоков
    ↓
Каждый процесс вызывает computeBC()
    ↓
  [GPU] bfs_kernel       — прямой BFS: вычисляет dist[], sigma[], stack[]
  [GPU] back_prop_kernel — обратное распространение: delta[], накопление в bc[]
    ↓
MPI_Reduce суммирует → rank 0 выводит результат
```

### Стратегия параллелизма CUDA

- **Один block = один исток**: потоки внутри блока обрабатывают рёбра параллельно
- Барьеры `__syncthreads()` между уровнями BFS гарантируют корректность чтения `sigma`
- Список предшественников не хранится: при обратном проходе предшественники восстанавливаются по условию `dist[v] == dist[w] - 1` (экономия памяти GPU)
- `atomicCAS` исключает повторное добавление вершины в очередь; `atomicAdd` обеспечивает безопасное накопление путей и зависимостей

### Формула центральности по посредничеству

$$BC(v) = \sum_{s \neq v} \sum_{t \neq s,v} \frac{\sigma(s,t \mid v)}{\sigma(s,t)}$$

В неориентированном графе каждая пара $(s,t)$ учитывается дважды, поэтому результат делится на 2.

## Результаты тестирования

Запуск на кластере IBM Polus (2 процесса MPI, по одному P100 на процесс), граф из 15 вершин:

```
Время вычислений (GPU BFS + обратное распространение): 0.0591 с
Время коммуникации (MPI_Reduce):                       0.0002 с
PASS — все 15 вершин: центральность верна (погрешность < 1e-06)
```

Накладные расходы на коммуникацию составляют всего **0,3%** от времени вычислений.

---

## Важные замечания

- Библиотека NCCL **не используется**: каждый GPU работает независимо, обмен данными только через MPI
- Код не содержит x86-специфичных intrinsics и полностью совместим с ppc64le
- `BATCH_SIZE` (по умолчанию 256) определяет число истоков в одном пакете; уменьшите при нехватке памяти GPU
